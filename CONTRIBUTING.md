# Contributing to survy

Thank you for your interest in contributing to **survy** 🎉
We welcome contributions of all kinds, including bug reports, feature requests, and code improvements.

---

## Getting Started

Clone the repository:

```bash
git clone https://github.com/hoanghaoha/survy.git
cd survy
```

Install in editable mode:

```bash
pip install -e .
```

---

## How to Contribute

### 🐛 Report Bugs

* Open an issue with a clear description
* Include steps to reproduce the problem
* Provide sample data if applicable

---

### 💡 Suggest Features

* Open an issue describing your idea
* Explain the use case and expected behavior

---

### 🔧 Submit Changes

1. Fork the repository
2. Create a new branch:

```bash
git checkout -b feat/your-feature-name
```

3. Make your changes
4. Commit your work (see commit guidelines below)
5. Push your branch and open a Pull Request

---

## Commit Message Guidelines

We follow a simple and consistent commit message convention:

```text
<type>: <short description>
```

### Types

* `feat`: New feature
* `fix`: Bug fix
* `chore`: Maintenance or tooling changes
* `test`: Add or update tests
* `docs`: Documentation changes
* `refactor`: Code changes without changing behavior
* `perf`: Performance improvements

---

### Examples

```text
feat: add support for compact multiselect parsing
fix: handle missing values in crosstab
docs: update README usage examples
chore: update CI workflow
```

---

### Guidelines

* Use lowercase for type
* Keep messages concise and clear
* Use present tense (e.g. "add", not "added")
* Focus on what changed, not how

---

## Coding Guidelines

* Keep code simple and readable
* Follow the existing API design
* Write clear and consistent function names
* Add docstrings where appropriate

---

## Testing

* Add tests for new features when possible
* Ensure existing functionality is not broken

---

## Project Scope

**survy** focuses on:

* Survey data processing and transformation
* Multiselect handling (compact and expanded formats)
* Clean and user-friendly data analysis workflows

Contributions aligned with this scope are especially welcome.

---

## Questions

If you’re unsure about anything, feel free to open an issue for discussion.
